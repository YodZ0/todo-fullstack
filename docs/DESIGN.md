# DESIGN.md — Минималистичный таск-менеджер с трекером времени

> Бриф для генерации UI в Stitch / Claude Design.
> Экраны описаны отдельными блоками — каждый можно вставлять в Stitch как самостоятельный промпт,
> сохраняя одинаковый блок «Дизайн-система» во всех промптах для консистентности.

---

## 1. Обзор продукта

Мобильное приложение — таск-менеджер (ToDo) для повседневного управления задачами:
создание, выполнение и удаление задач. Дополнительно — простой трекер рабочего времени,
который считает, сколько всего человек проработал за день, и статистика по этому времени.

Цель дизайна: спокойный, минималистичный интерфейс в мягкой кремовой палитре, который не отвлекает
и позволяет быстро добавить задачу, отметить выполненной и включить/выключить учёт времени.

## 2. Целевые пользователи _(предположение — уточни при желании)_

Люди, которые ведут личные и рабочие задачи и хотят видеть, сколько времени потратили на работу
за день. Пользуются в основном с телефона, короткими сессиями в течение дня.

## 3. Платформа

- Только **мобильное приложение** (телефон), портретная ориентация.
- Mobile-first, одноколоночная раскладка, крупные зоны нажатия.

---

## 4. Дизайн-система (использовать во всех экранах)

**Настроение:** минималистичный, чистый, спокойный, природный. Много «воздуха», мягкие
кремовые тона, шалфейно-зелёные акценты, контрастная чёрная кнопка действия.

**Темы:** светлая (кремовая) и тёмная с переключателем. Шалфейный акцент присутствует в обеих темах.

**Цвета (взяты из референса — кремово-шалфейная палитра):**

| Роль                                          | Светлая тема                | Тёмная тема                     |
| --------------------------------------------- | --------------------------- | ------------------------------- |
| Фон                                           | `#EFF0E2` (мягкий кремовый) | `#1A1B15` (тёплый почти-чёрный) |
| Поверхность/карточка                          | `#F9FAF0` (светлый крем)    | `#24261D`                       |
| Основной текст                                | `#1E1F19`                   | `#F1F2E6` (кремовый)            |
| Вторичный текст                               | `#6D7062`                   | `#9EA08E`                       |
| Границы/разделители                           | `#E2E4D3`                   | `#33352A`                       |
| Акцент (шалфей) — активное/выполнено/прогресс | `#8FA85E`                   | `#B4C09C`                       |
| Мягкий шалфей — фон пилюль/выделения          | `#CCD8B4`                   | `#3A3E2C`                       |
| Основная кнопка (primary)                     | `#191A15` (почти-чёрная)    | `#EDEFDF` (кремовая)            |
| Текст на основной кнопке                      | `#F2F3E8` (кремовый)        | `#191A15` (тёмный)              |

> Логика цвета: **primary-кнопка контрастная** (чёрная на светлой теме, кремовая на тёмной) —
> как чёрная кнопка «Buy now» в референсе. **Шалфейный зелёный** — вторичный акцент
> для выполненных задач, активных состояний и прогресса таймера.

**Типографика:** _(предположение)_ чистый нейтральный sans-serif (в духе Inter / SF Pro).
Крупные заголовки, комфортный body-текст, чёткая иерархия. Без декоративных шрифтов.

**Форма и пространство:**

- Просторные отступы, «воздух» между элементами (минимализм).
- Базовая сетка 8px.
- Скругления: карточки 16px, кнопки — pill/сильно скруглённые.
- Тени почти не используем — предпочитаем тонкие границы и мягкие поверхности.
- Минимальный набор цветов и элементов на экране.

**Компоненты:** строка задачи с чекбоксом, плавающая кнопка «+» (FAB, чёрная в светлой теме),
primary/secondary кнопки, крупный дисплей таймера, переключатель темы (toggle),
нижняя навигация (4 вкладки), карточки-графики для статистики.

**Тон текста в интерфейсе:** короткий, спокойный, по делу. Без лишних восклицаний.

## 6. Экраны

### Экран 1 — Список задач (главный)

Назначение: показать задачи на сегодня, дать быстро отметить выполненными, добавить новую.

- Заголовок экрана: «Задачи» + текущая дата.
- Список задач. Каждая строка: круглый чекбокс слева, текст задачи, у выполненных —
  зачёркнутый текст приглушённым цветом и заполненный шалфеем `#8FA85E` чекбокс.
- Свайп по задаче влево — удалить.
- Пустое состояние: спокойная иллюстрация/иконка + подпись «Задач пока нет».
- Плавающая кнопка «+» (FAB) — чёрная (`#191A15`) с кремовым плюсом, справа снизу —
  открывает нижний лист создания задачи (см. ниже).

### Нижний лист — Создание / редактирование задачи _(не отдельный экран)_

Открывается снизу (bottom sheet) поверх экрана 1, фон затемняется.

- Поле ввода названия задачи (в фокусе, клавиатура сразу открыта).
- _(Предположение)_ необязательная заметка. Держим минимально — без перегруза полей.
- Кнопки «Сохранить» (чёрная primary) и «Отмена».
- Свайп вниз или тап по затемнению — закрыть.

### Экран 2 — Трекер рабочего времени

Назначение: считать общее рабочее время за день. Механика: включил — секундомер идёт,
выключил — останавливается. За день все отрезки суммируются в часы + минуты.

- Крупный по центру дисплей секундомера (текущий запущенный отрезок), моноширинные цифры.
- Одна большая круглая кнопка Start / Stop:
  - в состоянии «остановлен» — чёрная кнопка «Старт»;
  - в состоянии «идёт» — контурная кнопка «Стоп» + шалфейный индикатор активности.
- Крупный блок «Сегодня: Ч ч ММ мин» — суммарное рабочее время за день.
- _(Предположение)_ ниже — список сегодняшних отрезков (время начала–конца, длительность).

### Экран 3 — Статистика (графики)

Назначение: показать динамику рабочего времени и выполненных задач.

- Столбчатый график рабочего времени по дням за неделю (высота столбцов в шалфейном `#8FA85E`).
- Сводка сверху: суммарное время за неделю, среднее в день.
- Блок по задачам: выполнено за период / всего (можно кольцевой прогресс или простой счётчик).
- Переключатель периода: Неделя / Месяц.
- Спокойные карточки на кремовом фоне, минимум линий сетки.

### Экран 4 — Настройки

- Переключатель темы: Светлая / Тёмная / Системная.
- _(Предположения, при желании убрать)_ уведомления, язык, «О приложении».

## 7. Аккаунты и данные

Регистрация и вход **не нужны** — вне рамок MVP. Данные хранятся локально на устройстве.
Экрана логина/регистрации в дизайне нет.

## 8. Обязательно / чего избегать

**Обязательно:**

- Достаточный контраст текста на кремовом фоне и на чёрных кнопках.
- Зоны нажатия не меньше 44×44 pt.
- Корректная работа обеих тем (кремовая светлая и тёплая тёмная).

**Избегать:**

- Визуального шума, множества цветов, тяжёлых теней и градиентов.
- Перегруженных форм и лишних полей.

## 9. Референсы

- **TickTick** — как основной ориентир по структуре и UX: чистые списки задач, круглые чекбоксы,
  светлая/тёмная темы, простой и функциональный интерфейс без лишнего.
- **Кремово-шалфейная эстетика** из приложенного референса (мягкие кремовые фоны, природные
  зелёные акценты, контрастная чёрная кнопка действия, много «воздуха»).
